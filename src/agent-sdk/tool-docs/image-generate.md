# Image Generate

`image-generate` creates synthetic bitmap visuals through the current KimiBuilt image path, persists usable outputs as session artifacts, and returns hosted URLs that downstream builders can embed.

Runtime path:

- Primary route: gateway-first OpenAI-compatible image generation via `/v1/images/generations`.
- Model default: configured GPT-image model, normally `gpt-image-2`.
- Provider handling: GPT-image request parameters are normalized for the current OpenAI image API; gateway-compatible `response_format` handling is preserved only where supported.
- Persistence: generated bytes are saved as reusable artifacts before the tool reports a successful image result.

Use it for:

- Website, landing page, dashboard, and HTML builds that need custom hero art, product scenes, textures, thumbnails, or subject-specific bitmap visuals.
- Document, PDF, PPTX, and report builds that need generated illustrations or synthetic visuals.
- Design variants when the user asks for options or when a visual build needs several selectable directions.

Prefer other image tools when:

- The user asks for real photography, current public figures, products, venues, news images, or sourced visuals. Use verified image URLs from search, Unsplash, or direct image ingestion instead.
- A vector diagram, chart, graph, or icon system is a better fit. Use `graph-diagram` or the project design/icon system.

Caller contract:

- Treat image generation as a build step, not instant prompt context. Allow up to several minutes for larger or multi-image requests.
- After the call, check `success`, `data.usableCount`, `data.artifacts`, `data.artifactIds`, and `data.markdownImages`.
- Continue into website, HTML, or document composition only after at least one reusable artifact or markdown image is present.
- If `usableCount` is `0`, or `artifacts` is empty for a downstream artifact build, stop and surface `diagnosticSummary` instead of embedding placeholders or stale image links.
- Use returned artifact/download/inline URLs directly; do not invent `/api/...` URLs or reuse raw provider payloads.

Batching:

- Default to one image.
- Use `n` with `batchMode:"auto"` for several versions of the same prompt.
- Use `prompts[]` with `batchMode:"parallel"` for distinct scene prompts.
- Keep batches to five images or fewer.
